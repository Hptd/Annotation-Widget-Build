Annotation Widget

前端页面标注说明系统 — 零代码侵入、独立运行、可开关、可拖拽绑定、跟随组件显隐。

1. 概述

Annotation Widget 是一个完全独立于项目代码的页面标注系统。任意前端项目只需引入一个 <script> 标签即可使用，无需修改原始项目源码。

典型的使用场景是原型评审：你在本地开发页面并添加标注（可编辑），评审者在局域网查看同一页面时只能浏览标注，不能修改。标注数据通过 Vite 中间件实现的 JSON 文件进行同步。

核心特点：零依赖、纯 ES6+、Shadow DOM 隔离、单文件 IIFE、localStorage 持久化、支持 JSON 导入导出。

4. 架构原理

4.1 注入与隔离

  1. 页面加载 annotation-widget.js（自执行的 IIFE）
  2. 脚本检测 DOMContentLoaded → 在 document.body 上挂载一个 <div> 作为 Shadow Host
  3. 在 Shadow Host 上调用 attachShadow({ mode: 'closed' }) 创建隔离 DOM
  4. CSS 样式字符串注入 Shadow DOM 内的 <style> 标签
  5. 所有 UI 元素（工具栏、编辑器、标记、面板、tooltip）均在 Shadow DOM 内创建和操作

4.2 元素定位策略

优先级策略说明
1绝对 XPath从目标元素向上遍历到 body，记录 tagName + 兄弟索引
2CSS 选择器（回退）id → class（过滤 data-v-* scoped class）→ nth-child

遇到有 id 的祖先节点则以此为锚点停止遍历。解析时 XPath 优先，CSS 选择器作为降级方案。

4.3 可见性追踪

  • MutationObserver 挂载到 document.body，监听 childList + attributes（style/class/hidden），变更后 200ms 防抖触发重检
  • IntersectionObserver 为每个元素绑定标注单独创建，检测视口交叉状态
  • 每收到 scroll/resize 事件，通过 requestAnimationFrame 重新计算所有标记位置

4.4 状态机

IDLE ──点图钉──> ANNOTATION_MODE
ANNOTATION_MODE ──点元素/空白──> EDITING
EDITING ──保存/取消──> ANNOTATION_MODE
ANNOTATION_MODE ──点图钉──> IDLE
任意状态 ──点列表──> PANEL_OPEN

4.5 事件总线

内部使用发布/订阅模式解耦模块通信：

ANNOTATION_ADD / UPDATE / DELETE / SELECT / REBIND
ANNOTATION_MODE_ON / OFF
STORAGE_CHANGED
PANEL_OPEN / CLOSE

4.6 数据模型

{
  id: "uuid",
  type: "element" | "position",
  elementSelector: { xpath, cssSelector, tagName, textContent } | null,
  position: { x: 0-100, y: 0-100 },      // 视口百分比
  offset: { top, left, placement },
  content: { text, createdAt, updatedAt, author, color },
  metadata: { url, viewport, pageTitle },

}

5. 构建打包

前置条件

确保项目已安装 esbuild：

npm install esbuild --save-dev

执行构建

node annotation-widget-build/build.cjs

输出：

[annotation-widget] Built: F:\...\public\annotation-widget.js (37.2 KB)

构建产物说明

产物位置用途
annotation-widget.jspublic/生产使用的单个 JS 文件，无任何外部依赖

build.cjs 做的事：

  1. 以 src/index.js 为入口，递归解析所有 import
  2. 将所有 ES 模块打包为一个 IIFE（立即执行函数）
  3. 目标 ES2020、UTF-8、minified
  4. 输出到 public/annotation-widget.js
注意：构建脚本命名为 build.cjs 而非 build.js，因为项目 package.json 中设置了 "type": "module"，esbuild 的 require() 语法需要用 CommonJS 扩展名。

6. 接入项目

构建产物 annotation-widget.js 是一个完全独立的文件，可以在任何前端项目中使用。

6.1 最简接入（仅本地单机使用，无局域网同步）

只需 1 步：

  1. 复制 annotation-widget.js 到项目的静态资源目录（如 public/、static/）
  2. 在 index.html 的 </body> 前添加一行：
<script src="/annotation-widget.js"></script>

完成。刷新页面即可看到右上角浮动工具栏。

6.2 完整接入（含局域网只读同步）

需要额外配置 Vite 中间件，步骤见第 7 节。

6.3 独立使用（任意 HTML 页面）

直接将 <script> 标签插入任意 HTML 文件即可，不需要任何构建工具。

<!DOCTYPE html>
<html>
<body>
  <h1>My Page</h1>
  <script src="./annotation-widget.js"></script>
</body>
</html>

7. 改造空白 Vite Vue 项目

从一个空白 Vite + Vue 项目开始，完整接入标注系统（含局域网只读同步）：

步骤 1：复制构建产物

cp annotation-widget.js your-project/public/

步骤 2：修改 index.html

在 </body> 前添加一行：

<script src="/annotation-widget.js"></script>

步骤 3：修改 vite.config.ts

添加中间件插件，在 plugins 数组中新增：

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import fs from 'fs'
import path from 'path'

const ANNOTATIONS_FILE = path.resolve(__dirname, 'public/annotations.json')

function annotationSyncPlugin() {
  return {
    name: 'annotation-sync',
    configureServer(server) {
      // 读取标注数据（局域网只读用）
      server.middlewares.use('/api/annotations', (req, res) => {
        if (req.method === 'GET') {
          try {
            if (fs.existsSync(ANNOTATIONS_FILE)) {
              const data = fs.readFileSync(ANNOTATIONS_FILE, 'utf-8')
              res.setHeader('Content-Type', 'application/json')
              res.end(data)
            } else {
              res.setHeader('Content-Type', 'application/json')
              res.end('[]')
            }
          } catch {
            res.statusCode = 500
            res.end('{"error":"read failed"}')
          }
          return
        }

        // 写入标注数据（仅 localhost 可用）
        if (req.method === 'POST') {
          const isLocalhost = req.headers.host?.startsWith('localhost') ||
                              req.headers.host?.startsWith('127.0.0.1')
          if (!isLocalhost) {
            res.statusCode = 403
            res.end('{"error":"read-only"}')
            return
          }
          let body = ''
          req.on('data', chunk => body += chunk)
          req.on('end', () => {
            try {
              fs.writeFileSync(ANNOTATIONS_FILE, body, 'utf-8')
              res.setHeader('Content-Type', 'application/json')
              res.end('{"ok":true}')
            } catch {
              res.statusCode = 500
              res.end('{"error":"write failed"}')
            }
          })
        }
      })
    }
  }
}

export default defineConfig({
  plugins: [vue(), annotationSyncPlugin()],
  server: {
    host: '0.0.0.0',  // 允许局域网访问
    port: 5173
  }
})
关键设计：中间件通过 req.headers.host 判断请求来源。localhost/127.0.0.1 可以 POST 写入，其他 IP 只能 GET 读取。这样评审者在局域网查看时自动获得只读权限。

完成

三步骤完成后，运行 npm run dev，即可在本地标注、局域网查看。

如果不是 Vite 项目？

构建工具等价操作

8. 用户使用教程

8.1 启动标注模式

  • 点击页面右上角浮动工具栏的图钉按钮（第一个）
  • 或按键盘快捷键 Ctrl+Shift+A

进入标注模式后，鼠标指针变为十字准星，图钉按钮高亮显示。

8.2 创建标注

  • 元素标注：点击页面上的任意元素（按钮、卡片、输入框等）→ 弹出文本编辑器
  • 位置标注：点击页面空白区域 → 创建自由浮动标注

8.3 编辑标注

  • 输入标注文字说明
  • 选择标记颜色（6 种预设色：蓝/红/绿/黄/紫/青）
  • 点击保存 → 标注标记出现在页面上
  • 点击取消 → 丢弃编辑
  • 点击删除 → 删除该标注

8.4 查看标注

  • 标注以编号圆形徽章显示在绑定元素右上角
  • 自由浮动标注显示为视口百分比位置的图钉徽章
  • 鼠标悬停徽章 → 显示标注文字预览 tooltip
  • 点击徽章（非标注模式下）→ 打开编辑器查看/修改

8.5 拖拽重绑定

  1. 在非标注模式下，鼠标按住标注徽章
  2. 拖拽到页面上另一个元素上 → 标注重新绑定到新元素
  3. 拖拽到页面空白区域 → 标注转为自由浮动

8.6 管理面板

点击工具栏的列表按钮（第二个）打开管理面板：

  • 搜索：输入关键词过滤标注列表
  • 编辑：点击"编辑"按钮打开该标注的编辑器
  • 定位：点击"定位"按钮，页面平滑滚动到标注目标并高亮闪烁
  • 删除：点击"×"按钮删除单个标注
  • 隐藏全部 / 显示全部：切换全局标注可见性
  • 导出 JSON：下载所有标注为 .json 文件
  • 导入 JSON：从 .json 文件导入标注
  • 清空全部：删除所有标注（需确认）

8.7 全局显隐

点击工具栏的眼睛按钮（第三个）一键切换所有标注的显示/隐藏。隐藏时眼睛图标变为斜线样式。

8.8 退出标注模式

  • 再次点击图钉按钮
  • 或再次按 Ctrl+Shift+A

8.9 局域网只读模式

当通过局域网 IP 地址（如 http://172.18.206.103:3006）访问页面时：

  • 图钉按钮不显示，无法进入标注模式
  • 标注列表仍可查看，但无编辑/删除/清空/导入按钮
  • 悬停和点击徽章仍可查看标注内容
  • 标注数据每隔 10 秒自动从 /annotations.json 拉取最新内容

9. 编程 API

标注系统挂载后，会在 window.__AnnotationWidget 上暴露以下 API：

方法说明
.enter()进入标注模式
.exit()退出标注模式
.toggle()切换标注模式
.showAll()显示全部标注
.hideAll()隐藏全部标注
.toggleVisibility()切换全局显隐
.openPanel()打开管理面板
.getAnnotations()获取当前所有标注数据
.destroy()销毁标注系统（移除 Shadow DOM 和事件监听）
.engine获取底层 engine 实例（高级用法）

示例：在浏览器控制台操作

// 获取所有标注
__AnnotationWidget.getAnnotations()

// 程序化进入标注模式
__AnnotationWidget.enter()

// 程序化打开管理面板
__AnnotationWidget.openPanel()

10. 常见问题

Q: 标注数据存在哪里？

本地 localhost：localStorage（key: annotation_widget_store）。同时如果配置了 Vite 中间件，每次变更也会写入 public/annotations.json。

局域网查看：从 /annotations.json 拉取，仅内存中使用，不写入任何地方。

Q: 局域网用户能编辑标注吗？

不能。中间件通过 req.headers.host 判断，非 localhost 的 POST 请求返回 403。widget 前端也会根据 location.hostname 隐藏编辑功能。

Q: 标注会跟随页面滚动吗？

会。元素绑定标注通过 getBoundingClientRect() 在每次 scroll/resize 时重新计算位置。自由浮动标注按视口百分比定位，天然适应窗口变化。

Q: 目标元素被删除了怎么办？

标注徽章会变成虚线问号样式（孤立状态）。你可以拖拽它到新元素上重新绑定。

Q: 支持 SPA 路由切换吗？

部分支持。MutationObserver 会检测 DOM 结构变化并重新检查所有标注。但如果路由切换导致整个页面区域被替换，某些标注可能变为孤立状态。当前版本最适用于同一个页面上的标注场景。

Q: 标注数据丢失了怎么办？

定期使用管理面板的"导出 JSON"功能备份标注数据。如果 localStorage 被清除，可以通过"导入 JSON"恢复。

Q: 如何完全移除标注系统？

在浏览器控制台执行：

__AnnotationWidget.destroy()

然后删除 index.html 中的 <script> 标签即可。

Q: 可以不显示浮动工具栏吗？

可以。调用 __AnnotationWidget.hideAll() 隐藏所有标注，工具栏仍会显示。如需完全隐藏整个系统，使用 __AnnotationWidget.destroy()。

Q: 支持移动端触屏吗？

工具栏和标注标记在移动端可正常显示和点击。拖拽重绑定功能在触屏上体验有限，建议在桌面端使用。

Q: 最多支持多少个标注？

理论上无上限。实测 500+ 标注无明显性能影响。所有 UI 在 Shadow DOM 内渲染，不触发 Vue 响应式系统。

Q: 为什么构建脚本叫 build.cjs 而不是 build.js？

因为项目 package.json 中 "type": "module" 使得 .js 文件被当作 ES 模块处理。esbuild 的 require() 语法需要用 CommonJS，所以用 .cjs 扩展名强制 CommonJS 模式。
